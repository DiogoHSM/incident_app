# Incidents Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the schema, queries, and three routes that let a team member declare an incident, see a filtered list of incidents, and view an individual incident's metadata page. No real-time, no timeline events, no role mutation, no status/severity transitions yet.

**Architecture:** New `incidents` and `incident_services` tables (forward-only Drizzle migration). All DB access goes through `src/lib/db/queries/incidents.ts`, which calls `requireTeamMember` from `@/lib/authz` so admins see all and members see only their teams' work — same pattern as services/runbooks. Three new App-Router routes under `src/app/(app)/incidents/`. No real-time machinery is added; the detail page is a static metadata view (timeline + SSE come in Plan 3, role/status mutations come in Plan 4).

**Tech Stack:** Next.js 16 (App Router) · TypeScript strict · Drizzle ORM 0.45 · Postgres 16 · Tailwind v4 · NextAuth v5 · Vitest 4 + testcontainers · zod 4 · pnpm.

**Spec reference:** `docs/superpowers/specs/2026-04-28-incident-tracker-design.md`. This plan implements §4.1 (Incident, IncidentService entities), §5.1 partial (`/incidents`, `/incidents/new`, `/incidents/[slug]`), §6.3 (incidents list filters). It defers §6.1 timeline + quick actions (Plan 3/4), §6.2 dashboard cards (Plan 4 or polish), and webhook-created incidents (Plan 6).

**Prerequisite folded in (Tasks 0a–0c):** Three blockers from `.claude/memory/foundation_followups.md` that the Plan 1 reviewer flagged before Plan 2 starts:
- Task 4 follow-up: testcontainer per-file boot will hurt at 5+ integration files. Switch to single shared container + transactional savepoints.
- Task 10 follow-up: `listServicesForUser` / `findServiceBySlugForUser` ignore admin-sees-all. Fix and add regression tests.
- Task 7 follow-up: `provisionUserOnSignIn` SELECT-then-INSERT race. Switch to `INSERT ... ON CONFLICT (email) DO UPDATE`.

**Language note:** All new content (code, tests, docs, comments, commit messages) is English. Existing PT-BR docs touched by this plan get translated as part of Task 10.

## Decisions encoded in this plan

1. **Slug format:** `inc-{8 chars base32url}` from `crypto.randomBytes(5).toString('base64url').replace(/[_=]/g, '').toLowerCase().slice(0, 8)`. Roughly 5 bytes = 40 bits ≈ ~10^12 space; collision risk for v1 traffic is negligible. No new dep. Generated client-side of the DB (in Node), retried up to 3× on `unique` violation.
2. **Initial status of manually-declared incidents:** `investigating`. `triaging` is reserved for webhook-created (Plan 6).
3. **Severity required at declaration.** Default in the form is SEV3 (matches the spec's "default to medium until you know better" tone, and keeps engineers from accidentally picking SEV1).
4. **Affected services at declaration:** zero or more. The set must be a subset of services the caller can see (their teams' services for members; all services for admins). Cross-team incidents (where affected services span teams) are allowed — they are owned by the team chosen at declaration time. This matches spec §3.6.
5. **IC / Scribe / Comms columns on the schema:** included as nullable now; no UI to set them in this plan (Plan 4).
6. **Timeline events table:** NOT added in this plan. Plan 3 owns it. Detail page renders header + affected services + runbooks only.
7. **Status / severity transitions:** NOT exposed in this plan. There is no "Mark resolved" button yet. Plan 4 owns it. Incidents in Plan 2 stay open indefinitely.
8. **Filters:** chip filters in the URL (`?status=...&severity=...&team=...&days=30`), parsed server-side. Default 30-day window.
9. **Test infra:** single Postgres container started in `globalSetup`. Each test runs inside `BEGIN ... ROLLBACK` via a `withTx(test)` helper. ~5× faster than per-file containers and the right shape going forward.

```ts
import { drizzle } from 'drizzle-orm/postgres-js';
import postgres, { type Sql } from 'postgres';
import { afterAll, beforeAll, beforeEach, afterEach, expect } from 'vitest';
import * as schema from '@/lib/db/schema';

export type TestDB = ReturnType<typeof drizzle<typeof schema>>;

interface Ctx {
  client: Sql;
  db: TestDB;
}

const ctx: Partial<Ctx> = {};

export function getTestDb(): TestDB {
  if (!ctx.db) throw new Error('Test DB not initialized — did you call useTestDb() in the suite?');
  return ctx.db;
}

const TABLES = ['runbooks', 'services', 'team_memberships', 'teams', 'users'] as const;

async function truncateAll(client: Sql): Promise<void> {
  await client.unsafe(`TRUNCATE TABLE ${TABLES.join(', ')} RESTART IDENTITY CASCADE`);
}

export function useTestDb(): void {
  beforeAll(async () => {
    const uri = process.env.TEST_DATABASE_URL;
    if (!uri) throw new Error('TEST_DATABASE_URL not set; check tests/setup/global.ts');
    ctx.client = postgres(uri, { max: 5 });
    ctx.db = drizzle(ctx.client, { schema });
  });

  beforeEach(async () => {
    if (ctx.client) await truncateAll(ctx.client);
  });

  afterAll(async () => {
    await ctx.client?.end();
  });
}

export const DB_ERR_UNIQUE = /duplicate|unique/i;
export const DB_ERR_FK = /foreign key|violates/i;
export const DB_ERR_NOT_NULL = /null value|not-null/i;

export function expectDbError(pattern: RegExp): object {
  return expect.objectContaining({
    cause: expect.objectContaining({ message: expect.stringMatching(pattern) }),
  });
}
```

Rewrite `src/lib/db/queries/services.ts`:

```ts
import { and, eq, inArray } from 'drizzle-orm';
import { type DB } from '@/lib/db/client';
import { services, type Service, type NewService } from '@/lib/db/schema/services';
import { teamMemberships } from '@/lib/db/schema/team-memberships';
import { findUserById } from '@/lib/db/queries/users';
import { requireTeamMember } from '@/lib/authz';

export async function listServicesForUser(db: DB, userId: string): Promise<Service[]> {
  const user = await findUserById(db, userId);
  if (!user) return [];
  if (user.role === 'admin') return db.select().from(services);

  const memberships = await db
    .select({ teamId: teamMemberships.teamId })
    .from(teamMemberships)
    .where(eq(teamMemberships.userId, userId));
  if (memberships.length === 0) return [];
  return db
    .select()
    .from(services)
    .where(
      inArray(
        services.teamId,
        memberships.map((m) => m.teamId),
      ),
    );
}

export async function findServiceBySlugForUser(
  db: DB,
  userId: string,
  slug: string,
): Promise<Service | null> {
  const user = await findUserById(db, userId);
  if (!user) return null;
  if (user.role === 'admin') {
    const [row] = await db.select().from(services).where(eq(services.slug, slug)).limit(1);
    return row ?? null;
  }
  const list = await listServicesForUser(db, userId);
  return list.find((s) => s.slug === slug) ?? null;
}

export async function createService(
  db: DB,
  callerId: string,
  input: Pick<NewService, 'teamId' | 'name' | 'slug' | 'description'>,
): Promise<Service> {
  await requireTeamMember(db, callerId, input.teamId);
  const [row] = await db.insert(services).values(input).returning();
  if (!row) throw new Error('Insert returned no rows');
  return row;
}

export async function updateService(
  db: DB,
  callerId: string,
  serviceId: string,
  patch: Partial<Pick<NewService, 'name' | 'description'>>,
): Promise<Service> {
  const [existing] = await db.select().from(services).where(eq(services.id, serviceId)).limit(1);
  if (!existing) throw new Error('Service not found');
  await requireTeamMember(db, callerId, existing.teamId);
  const [row] = await db.update(services).set(patch).where(eq(services.id, serviceId)).returning();
  if (!row) throw new Error('Update returned no rows');
  return row;
}
```

(Note: `findServiceBySlugForUser` for non-admins is intentionally still list-then-filter for now — fixing the N+1-ish query is in `foundation_followups.md` and not blocking. The `and(...)` import was unused — drop if your linter complains.)

```ts
import { sql } from 'drizzle-orm';
import { type DB } from '@/lib/db/client';
import { users, type User } from '@/lib/db/schema/users';

export interface ProvisionInput {
  email: string;
  name: string;
  ssoSubject: string;
  adminEmails: string[];
}

export async function provisionUserOnSignIn(db: DB, input: ProvisionInput): Promise<User> {
  const email = input.email.toLowerCase();
  const role = input.adminEmails.includes(email) ? 'admin' : 'member';

  const [row] = await db
    .insert(users)
    .values({ email, name: input.name, ssoSubject: input.ssoSubject, role })
    .onConflictDoUpdate({
      target: users.email,
      // Intentionally do NOT update role here — re-login should not demote/promote;
      // role transitions go through the admin UI.
      set: {
        name: sql`excluded.name`,
        ssoSubject: sql`excluded.sso_subject`,
      },
    })
    .returning();

  if (!row) throw new Error('Upsert returned no rows');
  return row;
}
```

```ts
import { teams } from './teams';
```

```ts
import { randomBytes } from 'node:crypto';

export function generateIncidentSlug(): string {
  const raw = randomBytes(6).toString('base64url').toLowerCase();
  // base64url uses [A-Za-z0-9_-]; lowercased that's [a-z0-9_-].
  // Strip non-alphanumerics to keep slugs clean, then take 8 chars.
  const clean = raw.replace(/[^a-z0-9]/g, '');
  return `inc-${clean.slice(0, 8).padEnd(8, '0')}`;
}
```

```ts
import { type DB } from '@/lib/db/client';
import {
  incidents,
  incidentServices,
  type Incident,
} from '@/lib/db/schema/incidents';
import { services } from '@/lib/db/schema/services';
import { findUserById } from '@/lib/db/queries/users';
import { teamMemberships } from '@/lib/db/schema/team-memberships';
import { requireTeamMember, ForbiddenError } from '@/lib/authz';
import { type Severity } from '@/lib/db/schema/services';
import { generateIncidentSlug } from '@/lib/incidents/slug';

export interface DeclareIncidentInput {
  teamId: string;
  title: string;
  summary: string;
  severity: Severity;
  affectedServiceIds: string[];
}

export async function declareIncident(
  db: DB,
  callerId: string,
  input: DeclareIncidentInput,
): Promise<Incident> {
  await requireTeamMember(db, callerId, input.teamId);
  await assertCanAttachServices(db, callerId, input.affectedServiceIds);

  return db.transaction(async (tx) => {
    let incident: Incident | undefined;
    for (let attempt = 0; attempt < 3 && !incident; attempt++) {
      const slug = generateIncidentSlug();
      try {
        const [row] = await tx
          .insert(incidents)
          .values({
            publicSlug: slug,
            teamId: input.teamId,
            declaredBy: callerId,
            severity: input.severity,
            title: input.title,
            summary: input.summary,
          })
          .returning();
        if (!row) throw new Error('Insert returned no rows');
        incident = row;
      } catch (e: unknown) {
        const msg = e instanceof Error ? e.message : '';
        const causeMsg =
          e instanceof Error && e.cause instanceof Error ? e.cause.message : '';
        const isUnique = /duplicate|unique/i.test(`${msg} ${causeMsg}`);
        if (!isUnique || attempt === 2) throw e;
      }
    }
    if (!incident) throw new Error('Could not allocate unique slug after 3 attempts');

    if (input.affectedServiceIds.length > 0) {
      await tx
        .insert(incidentServices)
        .values(input.affectedServiceIds.map((sid) => ({ incidentId: incident!.id, serviceId: sid })));
    }
    return incident;
  });
}

async function assertCanAttachServices(
  db: DB,
  callerId: string,
  serviceIds: string[],
): Promise<void> {
  if (serviceIds.length === 0) return;
  const user = await findUserById(db, callerId);
  if (!user) throw new ForbiddenError('Unknown user');
  if (user.role === 'admin') return;

  const myTeamIds = (
    await db
      .select({ teamId: teamMemberships.teamId })
      .from(teamMemberships)
      .where(eq(teamMemberships.userId, callerId))
  ).map((r) => r.teamId);

  if (myTeamIds.length === 0) throw new ForbiddenError('Cannot attach services');

  const allowed = await db
    .select({ id: services.id })
    .from(services)
    .where(inArray(services.teamId, myTeamIds));
  const allowedSet = new Set(allowed.map((s) => s.id));
  for (const id of serviceIds) {
    if (!allowedSet.has(id)) throw new ForbiddenError('Cannot attach service from another team');
  }
}
```

```ts
import { and, desc, eq, gte, inArray, sql } from 'drizzle-orm';
import { type Incident, type IncidentStatus } from '@/lib/db/schema/incidents';
// (merge with existing imports above; don't duplicate)

export interface ListIncidentFilters {
  status?: IncidentStatus;
  severity?: Severity;
  teamId?: string;
  daysBack?: number; // default 30
}

export async function listIncidentsForUser(
  db: DB,
  userId: string,
  filters: ListIncidentFilters,
): Promise<Incident[]> {
  const user = await findUserById(db, userId);
  if (!user) return [];

  const days = filters.daysBack ?? 30;
  const since = new Date(Date.now() - days * 24 * 60 * 60 * 1000);
  const conditions = [gte(incidents.declaredAt, since)];

  if (filters.status) conditions.push(eq(incidents.status, filters.status));
  if (filters.severity) conditions.push(eq(incidents.severity, filters.severity));

  if (user.role !== 'admin') {
    const myTeamIds = (
      await db
        .select({ teamId: teamMemberships.teamId })
        .from(teamMemberships)
        .where(eq(teamMemberships.userId, userId))
    ).map((r) => r.teamId);
    if (myTeamIds.length === 0) return [];

    const scopeTeams = filters.teamId
      ? myTeamIds.filter((t) => t === filters.teamId)
      : myTeamIds;
    if (scopeTeams.length === 0) return [];
    conditions.push(inArray(incidents.teamId, scopeTeams));
  } else if (filters.teamId) {
    conditions.push(eq(incidents.teamId, filters.teamId));
  }

  return db
    .select()
    .from(incidents)
    .where(and(...conditions))
    .orderBy(desc(incidents.declaredAt));
}
```

```tsx
interface Props {
  current: { status?: string; severity?: string; days?: string };
}

function chip(label: string, href: string, active: boolean) {
  return (
    <Link
      key={label + href}
      href={href}
      className={`rounded-full border px-3 py-1 text-xs ${
        active
          ? 'border-neutral-900 bg-neutral-900 text-white'
          : 'border-neutral-200 bg-white text-neutral-700 hover:bg-neutral-100'
      }`}
    >
      {label}
    </Link>
  );
}

function buildHref(curr: Props['current'], patch: Record<string, string | undefined>): string {
  const merged: Record<string, string | undefined> = { ...curr, ...patch };
  const params = new URLSearchParams();
  for (const [k, v] of Object.entries(merged)) {
    if (v) params.set(k, v);
  }
  const qs = params.toString();
  return qs ? `/incidents?${qs}` : '/incidents';
}

        {chip('all', buildHref(current, { status: undefined }), !current.status)}
        {INCIDENT_STATUS_VALUES.map((s) =>
          chip(s, buildHref(current, { status: s }), current.status === s),
```
